# Production (Azure) dependency set.
#
# Deliberately a subset of requirements.txt. With RETRIEVER_BACKEND=azure_ai_search
# and RERANKER_BACKEND=passthrough, the self-hosted retrieval stack is never
# imported, so its dependencies are not installed:
#
#   chromadb              -> replaced by Azure AI Search vector search
#   rank-bm25             -> replaced by Azure AI Search BM25
#   torch                 -> replaced by the Azure managed semantic ranker
#   sentence-transformers -> ditto
#   transformers          -> ditto
#
# Dropping torch alone takes the image from ~3 GB to ~300 MB, which is what
# makes Container Apps scale-to-zero cold starts tolerable: the platform must
# pull the image before serving the first request after a scale-up.
#
# retrieval/factory.py and retrieval/reranker_factory.py import their backends
# lazily, so nothing here is imported by accident. Keep versions in sync with
# requirements.txt.
